Hexo 搭建个人网站（基础教程）

前言

目前比较流行的有Jekyll，Hexo，Simple，Octopress，Pelican以及Lo·gecho等等。这些静态程序可以说都有各自的好处，但最后我选择了Hexo来搭建自己的博客

选择 Hexo 的原因主要有：

  • Hexo 基于 NodeJs 实现，在 Windows 上安装 NodeJs 开发环境简单
  • Hexo 可以通过简单的命令实现本地的预览，并直接发布到WEB容器中实现同步。
  • Hexo 的主题很多，也很不错
  • 支持 MarIDown 语法，用起来很方便

从网上查了一些资料，将搭建流程整理如下，其中有遇到的一些坑以及原理性的东西更便于理解。

流程：

  1. 安装Git Bash
  2. 安装NodeJs
  3. 安装hexo
  4. 生成SSH并添加到github
  5. 部署项目
  6. 上传到github
  7. 绑定个人域名
  8. 修改及配置主题

安装 Hexo

  • 创建一个目录用来存放你 blog 的文件，

  • 安装完成后查看版本

  • 注：<

    • 如果不成功查看本地是否添加ssh

      deploy:

      type: git
repo: git@github.com:YourgithubName/YourgithubName.github.io.git
branch: master

  • 回到命令行执行如下命令

    hexo clean
hexo generate
hexo server

    注：hexo 3.0把服务器独立成个别模块，需要单独安装：npm i hexo-server。

  • 打开浏览器输入：http://localhost:4000 就可以看见页面啦

上传到 GitHub

  • 又得安装啦：npm install hexo-deployer-git --save（这样才能将你写好的文章部署到github服务器上并让别人浏览到）

  • 执行命令(建议每次都按照如下步骤部署)：

    hexo clean
hexo generate
hexo deploy

    后两句可以合并 hexo g -d

  • 在浏览器中输入http://yourgithubname.github.io就可以看到你的个人博客啦，是不是很兴奋！

绑定个人域名

  • 购买域名，这里就不多说了
  • 在项目的 source 文件夹下新建一个名为CNAME的文件，在里面添加你购买的域名，比如我添加的是 mellete.com ，只能添加一个哦。
  • 获取IP ping yourname.github.io
  • 在 CDN列表添加一条记录指向你的仓库地址
  • 重新部署一次，用你的域名直接访问
  • 是不是很惊喜！！！
注：以上步骤后就可以写文章了，Markdown语法进行编写，在 \source\_posts 目录下新建 .md 格式的文件就可以了
