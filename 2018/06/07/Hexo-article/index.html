Hexo 搭建个人网站（文章编辑）

前言

目前比较流行的有Jekyll，Hexo，Simple，Octopress，Pelican以及Lo·gecho等等。这些静态程序可以说都有各自的好处，但最后我选择了Hexo来搭建自己的博客

前一篇已经讲述了 Hexo 的安装，配置，上传以及绑定自己的域名等步骤

接下来

马上要开始写文章了，Markdown语法进行编写，在 \source\_posts 目录下新建 .md 格式的文件

内容提要：

  1. 创建文章
  2. 文章属性 – 分类、标签、摘要
  3. 草稿
  4. 启动本地服务

开始啦 Go Go Go

创建文章

  • 在命令行输入 hexo new "my-article"
    之后在source/_posts目录下面，多了一个my-article.md的文件
  • 进入文件目录，打开文件进行编辑内容就好啦！！！
  • 推荐编辑器 typora

  • 格式说明：

    title: my-article
date: 2018-06-07 11:10:33
tag:

文章属性

  • 文件的开头是属性，采用统一的yaml格式，用三条短横线分隔。下面是文章正文。
    文章的正文支持markdown格式，建议你先学习一下它的语法。markdown不像html似的一大堆标签，很简单，只有几个符号。
Setting Description Default
layout Layout post或page
title 文章的标题
date 创建日期 文件的创建日期
updated 修改日期 文件的修改日期
comments 是否开启评论 true
tags 标签
category 分类
permalink url中的名字 文件名
  • 样例：
title: my-article
date: 2018-06-07 11:10:15
tags: 
    - Hexo
    - article
category: 
    - Hexo
    - 其他

  • 摘要：

    同wordpress一样，之上的内容为摘要。

  • layout

    如果你修改了layout，在scaffolds文件夹里一定要有名字对应的模版文件，否则会采用默认模版。

文章草稿

草稿相当于很多博客都有的“私密文章”功能。

hexo new draft "new draft"

在 source/_drafts 目录下,这个文件不被显示在页面上，链接也访问不到。也就是说如果你想把某一篇文章移除显示，又不舍得删除，可以把它移动到_drafts目录之中。

如果你希望强行预览草稿，更改配置文件：

render_drafts: true
或者，如下方式启动server：

hexo server --drafts
下面这条命令可以把草稿变成文章，或者页面：

hexo publish [layout] <filename>

启动本地服务

其实上面一已经提到如何启动了，
hexo server 就启动本地服务了，默认 http://localhost:4000/ 就可以访问预览你编写好的文章了，修改后只需要刷新就可以，是不是灰常方便啊~~~

到此，你基本上就可以完成整个博客的‘运营’了，剩下的就是一些美化，SEO以及小技巧等一些炫酷的东西啦，等待后续更新哦~~
